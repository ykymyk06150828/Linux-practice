# Linux コマンド研修アプリ 画面設計書

## 1. 文書概要

### 1-1. 文書名
Linux コマンド研修アプリ 画面設計書

### 1-2. 目的
本書は、Linux コマンド研修アプリにおける画面一覧、画面項目、画面遷移、表示内容、操作内容を整理し、画面実装および UI 設計の基礎資料とすることを目的とする。

### 1-3. 前提
- 本システムはブラウザ利用を前提とする
- 受講者画面と管理者画面を提供する
- ターミナル画面はブラウザ埋め込み型 UI を利用する
- UI は今後の研修内容・運用要件・ユーザビリティ改善に応じて改修される可能性がある
- 本書の画面設計は初期版の想定であり、今後変更される可能性がある

---

## 2. UI 設計方針

### 2-1. 基本方針
- シンプルで迷いにくい画面構成とする
- 受講者が Linux コマンド演習に集中できる UI とする
- ターミナル画面を主役とし、課題説明や操作ボタンを補助要素として配置する
- 管理者画面は一覧性と操作性を重視する
- PC ブラウザを主対象とするが、将来的なレスポンシブ対応も考慮する

### 2-2. UI 改修に関する方針
本システムの UI は、今後の利用状況や改善要望に応じて改修される可能性がある。  
そのため、画面設計では各表示領域を独立したコンポーネントとして整理し、レイアウト変更・情報追加・操作導線変更に柔軟に対応できる構成を前提とする。

### 2-3. 画面設計上の考慮事項
- ターミナル表示領域は再利用可能なコンポーネントとする
- 課題説明エリアは折りたたみや表示切替に対応しやすい構造とする
- ボタン配置は今後の追加・削除に対応しやすい設計とする
- 管理画面の一覧表示は列追加やフィルタ追加に対応しやすい構成とする

## 4. 画面遷移設計

### 4-1. 受講者画面遷移
1. ログイン画面  
2. 受講者トップ画面  
3. ターミナル演習画面  
4. 必要に応じてリセット確認ダイアログ表示  
5. ログアウト時にログアウト確認ダイアログ表示  
6. ログアウト後はログイン画面へ戻る  

### 4-2. 管理者画面遷移
1. 管理者ログイン画面  
2. 管理者トップ画面  
3. 受講者一覧画面 / コンテナ一覧画面 / ログ閲覧画面へ遷移  
4. 対象選択時に受講者詳細画面へ遷移  
5. 各種操作時に確認ダイアログ表示

## 5. 受講者向け画面設計

## 5-1. ログイン画面

### 画面概要
受講者がアプリへログインするための画面。

### 主な表示項目
- ロゴ / システム名
- ログイン ID 入力欄
- パスワード入力欄
- ログインボタン
- エラーメッセージ表示エリア

### 主な操作
- ログイン ID 入力
- パスワード入力
- ログイン実行

### 入力チェック
- 必須入力チェック
- 認証失敗時のエラー表示

### 備考
- 初回は最小限のシンプルな構成とする
- 将来的にパスワード再発行や SSO ボタン追加の可能性あり

---

## 5-2. 受講者トップ画面

### 画面概要
受講者がログイン後に最初に表示される画面。  
コース情報、課題概要、演習開始導線を表示する。

### 主な表示項目
- ヘッダー
- ユーザー名表示
- コース名
- 課題一覧または現在課題表示
- 課題説明サマリー
- 演習開始ボタン
- ログアウトボタン

### 主な操作
- 課題選択
- 演習開始
- ログアウト

### 備考
- 初期版では 1コース / 1課題前提でもよい
- 将来的に複数課題の進捗表示を追加できる構成とする

---

## 5-3. ターミナル演習画面

### 画面概要
Linux コマンド演習の中心となる画面。  
JSLinux 風の体験を意識しつつ、研修向けに管理しやすい構成とする。

### 画面構成案
- ヘッダーエリア
- 課題説明エリア
- ターミナル表示エリア
- 操作ボタンエリア
- ステータス表示エリア

### 主な表示項目
#### ヘッダーエリア
- システム名
- コース名
- 課題名
- ユーザー名
- ログアウトボタン

#### 課題説明エリア
- 課題タイトル
- 課題説明文
- ヒント表示欄
- 注意事項表示欄

#### ターミナル表示エリア
- xterm.js によるターミナル画面
- プロンプト表示
- コマンド入力欄相当領域
- 実行結果表示領域
- スクロール領域

#### 操作ボタンエリア
- 環境リセットボタン
- 接続再試行ボタン
- ヒント表示ボタン
- 課題一覧へ戻るボタン

#### ステータス表示エリア
- 接続状態
- 実行環境状態
- 最終操作時刻
- システムメッセージ

### 主な操作
- Linux コマンド入力
- 実行結果確認
- 環境リセット
- 接続再試行
- ヒント表示切替
- 画面遷移操作

### 画面上の注意点
- ターミナルエリアを最も大きく表示する
- 課題説明は固定表示または折りたたみ可能とする
- 接続エラー時は画面上部またはステータス欄にわかりやすく表示する
- ボタンは誤操作しにくい配置とする

### 備考
- UI は将来的に大きく見直す可能性がある
- 課題説明をサイドバー化する、上下分割にする、全画面ターミナル化する等の改修可能性を考慮する
- スマホ利用やタブレット対応が必要になった場合に備え、レイアウトは可変対応を意識する

---

## 9. レイアウト設計方針

### 9-1. 受講者画面
- ヘッダー固定
- メインエリアに課題説明とターミナル表示を配置
- ターミナルを最大の主要領域とする
- ステータスは目立ちすぎず確認しやすい位置に配置

### 9-2. 管理画面
- サイドメニューまたは上部タブで画面切替
- 一覧画面中心のシンプルな構成
- フィルタ / 検索エリアを追加しやすい設計

---

## 10. メッセージ設計方針

### 10-1. 成功メッセージ例
- ログインに成功しました
- 環境をリセットしました
- 再接続に成功しました

### 10-2. エラーメッセージ例
- ログイン ID またはパスワードが正しくありません
- ターミナル接続に失敗しました
- 環境のリセットに失敗しました
- セッションが切れました。再度ログインしてください

### 10-3. 確認メッセージ例
- 環境を初期状態に戻しますか？
- この受講者の環境を停止しますか？
- ログアウトしますか？

## 11. 今後の UI 改修可能性に関する記載

### 11-1. 改修可能性
本システムの UI は、今後の運用結果や研修内容の変化、利用者からのフィードバックを踏まえて改修される可能性がある。  
そのため、画面設計は現時点での初期想定を整理したものであり、正式実装時および運用後に見直される可能性がある。

### 11-2. 想定される改修例
- ターミナルと課題説明のレイアウト変更
- ダークモード対応
- スマホ / タブレット最適化
- 課題進捗表示の追加
- 管理ダッシュボードの高度化
- 操作ボタン配置の見直し
- ヒント表示や模範解答表示の追加
- 受講者向けナビゲーション改善

### 11-3. 設計方針
- コンポーネント分割を前提とする
- レイアウト変更がしやすい構造とする
- UI 変更時も API や業務ロジックへの影響を最小化する
- 初期版では機能優先、将来的に操作性改善を継続実施する

---

## 12. まとめ
本画面設計書では、Linux コマンド研修アプリに必要な受講者向け画面、管理者向け画面、共通画面を整理した。  
画面設計は初期版の想定であり、今後の要件追加や UI 改修に対応できるよう、部品化・疎結合・拡張性を意識した設計方針を採用する。
