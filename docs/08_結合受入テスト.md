## 3. 自動スモーク（API 疎通）

リポジトリ直下で（バックエンドが **3001** で待ち受けていること）:

```bash
chmod +x scripts/acceptance-smoke.sh
./scripts/acceptance-smoke.sh
```

**期待:** 終了コード 0。失敗時は `BASE_URL` または DB/Redis/バックエンド起動を確認する。

---

## 4. 負荷テスト（ベースライン）

**目的:** 同時多数からの API 到達性を確認する（**ブラウザ 30 タブの代替としての機械的ベースライン**）。本番同等の「30 人がターミナル操作」は **§5 の手動／記録**と併用する。

### 4-1. 前提

- [k6](https://k6.io/) をインストールする。

### 4-2. 実行例

```bash
# バックエンドが 127.0.0.1:3001 で起動していること
k6 run scripts/k6-api-baseline.js
```

**期待（スクリプト内しきい値）:** `http_req_failed` が **1% 未満**、P95 レイテンシが目安として **2s 未満**（環境により変動）。閾値外の場合はスケール・ネットワーク・同時実行数を見直す。

| 期待 | コンテナがユーザー単位で分離され、他ユーザーのホームが見えない（または別コンテナ ID）。 |
