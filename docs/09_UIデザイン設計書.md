# Linux研修アプリ UIデザイン設計書（参考画像反映版）

## 1. 目的

本アプリは、**ブラウザ上で Linux 環境を実行しながら学習できる研修アプリ**である。  
すでに機能は完成しているため、本フェーズでは **UIの見た目・情報整理・操作性の改善** を目的とする。

特に以下を重視すること。

- 受講者が演習に集中できること
- 管理者が一覧・登録・確認を効率よく行えること
- 長時間見ても目が疲れにくい画面であること
- 白を基調とした、モダンで清潔感のある UI にすること
- 今後の改修や画面追加にも耐えられる、統一感あるデザインにすること

---

## 2. 参考画像から取り入れる方向性

参考画像の「マイコース」画面をベースに、以下の要素をデザインに取り入れること。

### 2-1. 取り入れる要素
- 白ベースの落ち着いた画面
- カード型でコースを一覧表示する構成
- 余白を広めに取り、詰め込みすぎないレイアウト
- コンテンツエリアを中央寄せにして見やすくする構成
- ナビゲーションは上部でシンプルにまとめる構成
- 過剰な装飾を避け、コンテンツを主役にする見せ方

### 2-2. 改善する要素
参考画像の方向性は活かしつつ、以下は改善対象とする。

- カード内の情報量が増えても見やすい整理にする
- コース名、進捗、操作ボタンの優先順位を明確にする
- クリック可能な領域をわかりやすくする
- 画像だけに頼らず、情報設計でコースの違いが伝わるようにする
- リンクテキスト中心ではなく、ボタンやラベルで導線を明確にする
- 白ベースでも単調にならないよう、階層感を丁寧につける

## 3. デザインコンセプト

### コンセプト
**「白を基調に、学習と管理に集中できる、疲れにくいモダンUI」**

### キーワード
- White / Clean
- Modern / Minimal
- Calm / Comfortable
- Practical / Usable
- Structured / Readable

### デザインの基本姿勢
- 見た目の派手さより、使いやすさを優先する
- コントラストは十分に保ちつつ、刺激が強すぎない配色にする
- 長時間利用でも目が疲れにくい余白・文字サイズ・色設計にする
- 研修アプリとして、安心感と学習しやすさを感じる UI にする

## 4. 全体トーン＆マナー

## 4-1. 目指す印象
- 清潔感がある
- 情報が整理されている
- 見やすい
- 落ち着いている
- エンジニア学習サービスらしい知的な印象がある
- モダンだが冷たすぎない

## 4-2. 避けるべき表現
- 真っ白すぎて境界が見えないデザイン

- 文字量に対して余白が足りず窮屈な画面
- 学習画面なのに装飾が強すぎて集中を妨げる UI

## 5. カラーデザイン方針

## 5-1. ベースカラー
白を基調にした、目が疲れにくい配色を採用すること。

- メイン背景：やや柔らかい白
- セクション背景：白またはごく薄いグレー
- カード背景：白
- 枠線：薄いグレー
- 文字色：やや柔らかいダークグレー
- 補助文字：中間グレー

### カラーイメージ
- 背景：#FAFAFB ～ #F7F8FA
- カード：#FFFFFF
- 境界線：#E5E7EB 前後
- 本文：#1F2937 前後
- 補助テキスト：#6B7280 前後

## 5-2. アクセントカラー
アクセントは最小限に絞ること。  
基調は白だが、**選択中・進行中・重要アクション** が埋もれないように、1つの主アクセントカラーを定める。

推奨方向性：
- 落ち着いた青系
- 青緑系
- もしくは参考画像のサービスカラーに近い暖色アクセントを限定利用

### ルール
- 主ボタン
- アクティブタブ
- 選択中状態
- リンクや強調ポイント

にのみ重点的に使う。

## 5-3. ステータスカラー
- 成功：やわらかい緑
- 警告：やわらかい黄
- エラー：やわらかい赤
- 情報：やわらかい青

彩度が高すぎる色は避け、白背景に馴染むトーンにすること。

## 6. タイポグラフィ方針

## 6-1. 文字の役割分け
- ページタイトル
- セクション見出し
- カードタイトル
- 本文
- 補助テキスト
- ラベル
- テーブル内テキスト
- ターミナル / ログ用等幅フォント

## 6-2. 方針
- 可読性を最優先する
- 長時間見ても疲れにくい文字サイズにする
- 白背景で文字が細すぎないようにする
- 英数字、Linux コマンド、ログ表示は等幅フォントを使い分ける

## 6-3. 推奨の見え方
- ページタイトル：大きめ、太め
- セクションタイトル：中程度、太め
- 本文：標準サイズ
- 補助テキスト：少し小さめ
- カード内の補足情報：小さすぎず、読み捨てにならないサイズ
- ターミナル：視認しやすい等幅フォント

## 7. 余白・レイアウト方針

## 7-1. 基本方針
このアプリでは **余白がデザイン品質そのもの** になる。  
白ベースの UI は、余白不足だと一気に見づらくなるため、要素間の間隔を意図的に広めに取ること。

## 7-2. ルール
- ページ外周に十分な余白を取る
- セクションごとに明確な余白を設ける
- カード間の横・縦余白を統一する
- テーブルやフォームの上下余白も一定ルールに揃える
- 1画面に情報を詰め込みすぎない

## 7-3. コンテンツ幅
- 極端に横に広がりすぎないよう、最大幅を設定する
- 中央寄せのコンテンツ領域を基本にする
- ダッシュボードやコース一覧では、視線移動が長くなりすぎない横幅に調整する

---

## 8. 全体レイアウト設計

## 8-1. 共通レイアウト
- 上部：ヘッダー / グローバルナビゲーション
- 中央：メインコンテンツ
- 必要に応じて左サイドナビまたは上部タブ
- コンテンツは白背景ベースで、セクション単位に整理

## 8-2. ヘッダー
### 方針
参考画像のように、**ヘッダーはシンプルで軽く見せる**。

### 要件
- 左：ロゴまたはアプリ名
- 中央または左寄り：主要メニュー
- 右：通知、ユーザ名、プロフィールメニュー、ログアウト
- 高さは抑えめにして圧迫感をなくす
- アクティブなメニューは下線または背景差で明確にする

## 8-3. ナビゲーション
- 管理者と受講者で表示項目を切り替える
- 現在位置が一目でわかること
- クリック領域を十分確保する
- アイコンは必要最小限にする
- 派手な装飾ではなく、白背景に馴染む表現にする

## 9. ユーザ別 UI 方針

## 9-1. 管理者向け
### 重視すること
- 一覧性
- 検索性
- 編集効率
- 状態把握のしやすさ
- 誤操作防止

### デザイン方針
- テーブル中心
- 情報密度はやや高めでもよいが、余白で整理する
- 主要操作はページ上部に固定的に置く
- フィルタ、検索、ソートを自然に使える UI にする

## 9-2. 受講者向け
### 重視すること
- 学習導線のわかりやすさ
- 迷わないこと
- 演習への集中
- 進捗がわかること
- 続きから再開しやすいこと

### デザイン方針
- テーブルではなくカードやステップ構造を積極的に活用
- 余計な情報を出しすぎない
- 1画面1目的を意識する
- コース → タスク → 演習 の流れを自然につなぐ

## 10. マイコースカード設計（参考画像反映）

本アプリの受講者向け「マイコース」のカードは、参考画像の方向性を踏襲しつつ、以下の仕様で改善すること。

## 10-1. 基本方針
- 白背景の画面に馴染む、軽やかなカードにする
- コース一覧はカード形式で表示する
- カードの情報は「見るだけ」でなく「次の行動につながる」構造にする
- 画像ありでも画像に依存しすぎず、文字情報と操作導線を整理する

## 10-2. カード構造
カードは以下の縦構成を基本とする。

1. サムネイル画像
2. コース名
3. 進捗情報
4. ステータスラベル
5. 主操作ボタン

## 10-3. 見た目のルール
- 背景は白
- 薄い境界線をつける
- 影はごく弱くする
- 角丸は中程度
- カード間の余白は広めにとる
- ホバー時は少し浮く程度の変化に留める
- クリックできることが自然に伝わる表現にする

## 10-4. サムネイル画像
- カード上部いっぱいに表示
- 横長で統一
- 画像がなくても破綻しないプレースホルダを用意
- 画像上に文字を重ねすぎない
- コース内容の世界観を補助する程度に使う

## 10-5. コース名
- カード内で最も目立つ情報にする
- 1～2行程度で収める
- 長い場合は省略しつつ、全文確認導線を用意する
- 文字リンク風ではなく、カードタイトルとして見せる

## 10-6. 補足情報
補足情報は原則としてカード内に多く載せすぎない。  
**担当講師、対象者、学習期間の目安は表示しない。**

カード内では、受講者が今すぐ必要とする情報だけに絞ること。

表示対象は以下を優先する。

- 受講状態
- 進捗率
- 次に取るべき行動

詳細情報が必要な場合は、カード遷移後のコース詳細画面で確認できるようにする。

## 10-7. 進捗表示
カード内には進捗が必須。

表示要素：
- 進捗率（%）
- 進捗バー
- 未着手 / 受講中 / 完了 の状態ラベル

進捗が一目でわかることで、受講者が「次に何をやるか」を判断しやすくする。

## 10-8. 主操作
カード下部に、状態に応じた主操作を表示する。

例：
- 開始する
- 続きから学習
- 詳細を見る
- 完了を確認

リンクテキストではなく、**押しやすいボタン** で表現すること。

## 10-9. カード一覧レイアウト
- PCでは 2～4 列で表示
- 画面幅に応じて自動で折り返す
- 横幅いっぱいに引き伸ばさず、見やすいカラム幅を保つ
- 余白を十分に確保し、ギャラリーのような見せ方にする

## 14. レスポンシブ方針

## 14-1. PC
- 主利用環境
- 受講者画面はカード一覧と 2 カラム演習画面を基本とする
- 管理画面は一覧性を重視する

## 14-2. タブレット
- カード列数を減らす
- サイドメニューは折りたたみ対応
- 演習画面は上下レイアウトに変更可能にする

## 14-3. スマホ

## 15. AI実装時の具体指示

以下の方針で実装すること。

### 必須要件
- 白を基調とした、目が疲れにくい配色で設計すること
- 余白を十分に取り、窮屈にしないこと
- 参考画像の「マイコース」カードの方向性を取り入れること
- ただしリンク一覧のような見せ方ではなく、進捗と操作導線を整理したカードに改善すること
- カード内に担当講師、対象者、学習期間の目安は表示しないこと
- 管理画面はテーブル中心で、見やすく効率的にすること
- 受講者画面はカード中心で、学習導線を明確にすること
- 演習画面は最優先で使いやすく設計すること
- コンポーネントの見た目と余白ルールを共通化すること
- 今後の改修に耐えるよう、デザインシステムとして実装すること

### 禁止事項
- 色数を増やしすぎること
- 派手なグラデーションや強い装飾を多用すること
- 白背景なのに境界が見えなくなること
- 演習画面のターミナル領域を狭くすること
- カード内に情報を詰め込みすぎること
- テキストリンクだけで主要操作を済ませること
